# 搜索逻辑优化计划

## 目标
修改搜索功能，从"实时搜索"改为"按钮触发搜索"，避免每输入一个字符就触发搜索，提升性能和用户体验。

## 当前问题
- 每输入一个字符就触发搜索，对于大数据集会导致性能问题
- 频繁的搜索操作影响用户体验

## 修改方案

### 1. 移除实时搜索监听
- 文件：`src/gui/table_view.py`
- 移除 `<KeyRelease>` 事件绑定
- 保留搜索框输入功能

### 2. 添加搜索按钮
- 在搜索框旁边添加"搜索"按钮
- 点击按钮或按回车键触发搜索
- 搜索按钮样式：主要按钮

### 3. 保持清除按钮
- 保留现有的清除按钮
- 清除时同时清空搜索框和搜索结果

## 实施步骤
1. 修改 `table_view.py` 的 `_setup_ui` 方法，移除 `<KeyRelease>` 绑定
2. 添加搜索按钮
3. 绑定回车键触发搜索
4. 测试搜索功能

## 影响范围
- `src/gui/table_view.py` - 搜索 UI 和逻辑
- 不影响其他模块

## 预期结果
- 用户输入时不会触发搜索
- 点击"搜索"按钮或按回车键才执行搜索
- 提升大数据集的响应速度
